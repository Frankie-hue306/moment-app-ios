## 完整代码审查（第二次审查 v279）
### 🔴 严重问题（6个）
1. **`nextId` 计算bug** — 用 `Date.now()` 做用户ID，和 `nextId` 自增ID混用，可能产生ID冲突
2. **Token 安全漏洞** — 登录时生成新token但旧token不过期，多设备登录token仍有效
3. **前端 `D.m` 和服务器数据不同步** — 离线缓存数据 `D.m` 只在 gallery 刷新时部分覆盖，服务器删除动态后本地仍显示
4. **`/api/image/:id` 双重读 db.json** — 每次请求都重新读整个 db.json，高并发下性能差
5. **Nginx 缺少 WebSocket 支持** — 如果未来需要 WebSocket 功能，当前配置不支持
6. **avatar 用 dataUrl 存** — 图片 avatar 仍然是 base64 存 json，没有迁移到文件存储

### 🟡 中等问题（8个）
1. **`isDev` 绕过验证码逻辑** — 开发模式下任何人输入任意6位数字即可登录（非000000即可）
2. **auth 中间件 userId 解析** — 从 `req.body.userId` 或 `req.query.userId` 取，建议用数字 id 而不是让客户端传入
3. **Gallery `forceRefresh` 缺失** — 切换页面时 gallery 不会重新加载，只有手动下拉刷新
4. **`/api/gallery` 返回字段不一致** — 返回 `dataUrl` 但前端期望 `imageUrl`
5. **侧边栏昵称不同步** — `editNickname` 只更新 localStorage 和服务端，但不更新 `updateSideMenuUser`
6. **sw.js 缓存策略过于简单** — cache-first 策略可能缓存过期内容，建议 network-first
7. **Nginx 缺少 gzip 压缩** — 前端资源较大（143KB），没有 gzip 压缩
8. **404 页缺失** — 访问不存在的路径返回 Cannot GET /，没有兜底

### 🟢 轻微问题（5个）
1. **`manifest.json` theme_color 不一致** — 设置为 `#FF6B35` 但 app 主题色是 `#D4A373`
2. **landing.html 的 TestFlight 链接** — 指向占位链接 `your-beta-code`
3. **`tokenCreatedAt` 未校验** — 旧用户没有 tokenCreatedAt 字段，登录后才会设置
4. **`onboardNext` 用百分比硬编码** — 3页引导页用 `100/3`，增加页面需要改
5. **`nohup` 方式启动无日志轮转** — 部署脚本的 nohup 方式无日志轮转
