# 🤖 Agent 团队工作手册 v2.0

## 团队配置（13人）

```
📊 分析师团队 (4人)
├── market-analyst        技术面+K线+量价
├── fundamentals-analyst  财务+ROE+护城河
├── news-analyst          新闻+公告+政策
└── social-analyst        龙虎榜+情绪+资金

🔬 研究辩论团队 (3人)
├── bull-researcher       多方：找5个看涨理由
├── bear-researcher       空方：找5个看跌理由
└── research-manager      综合裁决偏多/偏空/中性

⚖️ 风险评估团队 (3人)
├── safe-analyst          保守：最差跌多少
├── risky-analyst         激进：最好涨多少
└── neutral-analyst       中性：概率加权EV

💰 决策团队 (3人)
├── trader                交易计划+价位
├── risk-manager          仓位+止损+否决
└── portfolio-manager     组合平衡+配置
```

## 每日工作流程

```
🕒 15:00 收盘
🕘 次日 08:47 推送（唯一推送时间）

08:47 执行步骤：
  第一步：获取昨日大盘数据
  第二步：四维模型筛选10只关注池
    维度1 = 昨日Top20热度（成交额/涨幅榜/龙虎榜）
    维度2 = 主力资金（大单净买/北向/机构席位）
    维度3 = 游资动向（龙虎榜营业部/一线游资）
    维度4 = 近一周新闻（公告/政策/订单/产品）
    必须命中≥3维，覆盖价值+成长+周期+防御
    每只标注大师框架（巴菲特/林奇/格雷厄姆/马克斯/费雪）

  第三步：筛查隔夜变化
    - 美股收盘（纳指/道指/费城半导体）
    - 每只个股隔夜公告（减持/回购/澄清/中标）
    - 宏观事件（美联储/政策/IPO/地缘）
    重新排序：🔥升级 / 无标记 / ⚠️降级 / ❌退榜

  第四步：添加框架评级
    🟢偏多 = 多维共振+隔夜利好
    🟡观望 = 信号不明，等右侧确认
    🔴偏空 = 利空为主，框架看淡

  第五步：推送到手机（Python JSON POST 方式）
```

## 推送技术规范

```python
import urllib.request, json, time

stocks = [
    "6.12 🔥🟢1.股票名代码 核心理由(隔夜:利好简述)",
    "6.12 🟢2.股票名代码 核心理由(隔夜:无变化)",
    "6.12 🟡6.股票名代码 核心理由(等右侧确认)",
    "6.12 🔴9.股票名代码 退榜理由(隔夜:利空简述)",
    "⚠️以上评级为框架分析 不构成投资建议"
]

for s in stocks:
    data = json.dumps({"title": s, "body": "长按复制代码", "autoCopy": "1"}).encode()
    req = urllib.request.Request(
        "https://api.day.app/ajB788VdEQVvkE5aRMGCjd",
        data=data, headers={"Content-Type": "application/json"}
    )
    urllib.request.urlopen(req)
    time.sleep(0.5)
```

⚠️ 禁止事项：
- 禁止使用 group 参数（会导致 iOS 折叠通知）
- 禁止在 URL 中放中文（会导致编码错误）
- 禁止用 curl 推中文（同上）
- 推送间隔必须 ≥0.5 秒
